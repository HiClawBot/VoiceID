VoiceID usage docs

VoiceID is a voice identity protocol. These docs describe how the Browser Playground demonstrates voice challenges, simulated proofs, test wallet signatures, and local session output; it is not an online authentication system.

Status: Browser Playground v0.1 Scope: demo-only / no server authentication Ports: 3400-3499

The Chinese visual decision brief shows the dependency path, risks, and release gates for a real Online Beta. Detailed source material remains in the Markdown implementation plan.

Overview

VoiceID aims to connect voice signals to an auditable identity protocol. The Playground only demonstrates UI and object flow. It does not touch mainnet assets, identify a speaker, perform real identity checks, or persist raw audio. 

Open the Playground
  -> Generate a random voice challenge
  -> Capture browser audio and run local quality checks
  -> Create a simulated VoiceProof
  -> Bind a test wallet address
  -> Sign a test message
  -> Return an AuthSession

Quickstart

The current repository is a static GitHub Pages site. Local preview does not require dependency installation.

Local preview 
python3 -m http.server 3400

Open the local site:

http://localhost:3400/

Only ports 3400-3499 are allowed. If 3400 is busy, use 3401, then 3402.

Playground usage

The Chinese-language Browser Playground is available at /beta/. It exposes this local demo flow:

  1. Display a random digit challenge.
  2. Request microphone access and start capture.
  3. Check volume, duration, and input availability.
  4. Create a simulated VoiceProof.
  5. Connect a test wallet.
  6. Sign a test message and return an AuthSession.

If microphone permission is denied, the page must show a recoverable error instead of blocking the flow.

The repository also includes the Chinese ADR-001, ADR-002, Protocol Schema, and acceptance checklist.

Voice proof

VoiceProof is a demo-only Playground object. It only means an audio-quality threshold passed; it does not establish speaker identity, and raw audio is not persisted.

VoiceProof example 
{
  "id": "vp_a62e15d4",
  "type": "VoiceProof",
  "version": "0.1",
  "challengeId": "vc_7f3c9a21",
  "scope": ["login", "wallet:sign"],
  "createdAt": "2026-07-11T08:00:18.000Z",
  "expiresAt": "2026-07-11T08:05:18.000Z",
  "method": "local-simulated-v0",
  "assurance": "demo-only",
  "status": "verified",
  "audioQuality": {
    "durationMs": 4218,
    "rms": 0.084,
    "peak": 0.612,
    "clippingRatio": 0.001,
    "quality": "good",
    "source": "microphone"
  },
  "rawAudioPersisted": false
}

Wallet signing

The Playground only permits test signatures. It must not trigger transfers, token approvals, delegated spending, or mainnet asset operations.

VoiceID Playground Login
Domain: localhost:3400
Nonce: 5b8f31a2d0e4
VoiceProof: vp_a62e15d4
Scope: login,wallet:sign
Expires: 2026-07-11T08:05:18.000Z

Missing wallet, rejected connection, and rejected signature must each return a distinct error state.

Security boundary

  • Do not persist raw audio.
  • Do not initiate real transfers.
  • Do not present simulated voice proof as production biometrics.
  • Every proof object must have an expiration time.
  • Users must be able to recover after denying permissions or signatures.

Developer integration

This is a non-executable sketch of a future SDK, not an API exported by the current repository. The native modules in beta/adapters/ are the executable source of truth.

Future interface sketch · not executable 
const voiceid = await VoiceID.connect({
  network: "testnet",
  wallet: "injected"
});

const session = await voiceid.authenticate({
  scope: ["login", "wallet:sign"],
  proof: "voice"
});

await session.verify();
