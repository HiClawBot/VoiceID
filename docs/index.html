VoiceID 使用文档

VoiceID 是声纹身份协议。当前文档说明 Browser Playground 如何演示声音挑战、模拟证明、测试钱包签名和本地会话输出；它不是线上认证系统。

状态：Browser Playground v0.1 范围：demo-only / 无服务端认证 端口：3400-3499

真实 Online Beta 的依赖路径、风险与发布门见 可视化决策稿；完整细节保留在 Markdown 施工计划。

概览

VoiceID 的目标是把声音信号接入可审计的身份协议。当前 Playground 只证明界面与对象流转，不直接处理主网资产，不做实名认证或说话人识别，不长期保存原始音频。 

用户打开 Playground
  -> 生成随机声纹挑战
  -> 浏览器采样并完成本地质量检测
  -> 生成模拟 VoiceProof
  -> 绑定测试钱包地址
  -> 签名测试消息
  -> 输出 AuthSession

快速开始

当前仓库是静态 GitHub Pages 项目，本地预览不需要安装依赖。

本地预览 
python3 -m http.server 3400

打开本地地址：

http://localhost:3400/

端口只能使用 3400-3499。如果 3400 被占用，依次使用 3401、3402。

Playground 使用

Browser Playground 已在 /beta/ 提供本地模拟闭环。它按下面顺序工作：

  1. 显示随机数字挑战。
  2. 请求麦克风权限并开始采样。
  3. 检测音量、时长和输入可用性。
  4. 生成模拟 VoiceProof。
  5. 连接测试钱包。
  6. 签名测试消息并输出 AuthSession。

如果浏览器拒绝麦克风权限，页面必须显示可恢复错误，不应卡死。

实现边界与字段定义见 ADR-001、 ADR-002、 Protocol Schema 和 Beta 验收清单。

声纹证明

VoiceProof 是 Playground 的 demo-only 证明对象。它只表达音频质量阈值通过，不代表说话人身份；原始音频不会被持久化。

VoiceProof 示例 
{
  "id": "vp_a62e15d4",
  "type": "VoiceProof",
  "version": "0.1",
  "challengeId": "vc_7f3c9a21",
  "scope": ["login", "wallet:sign"],
  "createdAt": "2026-07-11T08:00:18.000Z",
  "expiresAt": "2026-07-11T08:05:18.000Z",
  "method": "local-simulated-v0",
  "assurance": "demo-only",
  "status": "verified",
  "audioQuality": {
    "durationMs": 4218,
    "rms": 0.084,
    "peak": 0.612,
    "clippingRatio": 0.001,
    "quality": "good",
    "source": "microphone"
  },
  "rawAudioPersisted": false
}

钱包签名

Playground 只允许测试签名，不允许转账、授权代扣或主网资产操作。签名消息必须让用户看懂域名、nonce、用途和过期时间。

VoiceID Playground Login
Domain: localhost:3400
Nonce: 5b8f31a2d0e4
VoiceProof: vp_a62e15d4
Scope: login,wallet:sign
Expires: 2026-07-11T08:05:18.000Z

钱包连接失败、用户拒绝签名、钱包不存在都必须成为独立错误状态。

安全边界

  • 不保存原始音频。
  • 不发起真实转账。
  • 不把声纹证明当作生产级生物识别结果。
  • 所有证明对象必须有过期时间。
  • 用户拒绝权限或签名时必须可回到上一步。
