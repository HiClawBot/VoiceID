01 / Verdict

现在能发布什么，不能承诺什么

当前仓库已是一个边界清楚的本地协议演示，但它没有真实语音识别、服务端验签、会话撤销或生物数据治理能力。

执行判断

立刻发布 A，立即施工 B；C 保持门控。任何客户端生成的分数、签名结果和会话都不能成为认证事实，直到服务器完成一次性挑战消费与密码学验证。

当前已经证明

  • 麦克风授权、音频质量计算与音轨释放流程。
  • EIP-1193 钱包连接与可读测试签名请求。
  • demo-only VoiceProof、session-only 档案和静态发布检查。
  • 官网、文档、品牌资产具备继续演进的基础。

当前没有证明

  • 用户念对挑战或声音属于已注册用户。
  • 音频不是回放、合成或虚拟设备注入。
  • 钱包签名有效且 nonce 未被重放。
  • 声纹处理满足同意、删除与安全要求。

03 / Units

三个发布单元，各自只承诺能证明的事实

把静态体验、真实认证与声纹试点拆开，避免模拟路径与生产信任边界互相污染。

1–2 天

A · Browser Playground v0.1

发布现有体验、Docs 与决策稿。声音只做本地质量演示；钱包签名不经服务端验证。

退出信号：公网 200、CI 绿色、无误导文案。
4–6 周

B · Online Beta v0.2

Passkey 或服务端 SIWE 建立真实会话；语音挑战与评估可运行，但默认不参与放行。

退出信号：邀请制用户可注册、登录、退出、撤销、删除。
6–10+ 周

C · Verified Voice Pilot

只在受控设备、人群和数据区域内试验短语匹配、说话人匹配与 PAD 风险信号。

退出信号：模型、攻击、隐私和独立安全审查全部通过。
04 / Trust

认证信任根必须位于服务器

浏览器负责交互和采集，不负责宣布认证成功。声音输出可解释的策略信号；服务端强认证仍是会话成立的必要条件。

不可信输入浏览器 / DOM / 客户端分数

可被篡改；本地会话与 serverVerified 值不构成事实。

→
信任根Passkey 或服务端 SIWE

服务器校验 origin、challenge、nonce、签名、时间与单次消费。

→
附加信号VoiceAssessment

分别输出质量、短语匹配、说话人相似度与 PAD 风险。

→
可信结果服务端会话

HttpOnly、可轮换、可撤销、带 idle 与 absolute expiry。

05 / Architecture

小型模块化后端，加一个隔离的语音运行时

首版不引入合约、DID 节点、事件总线或服务网格。除敏感语音推理隔离外，其余后端保持单一模块化服务。

06 / Flows

先完成强认证，再接语音垂直切片

语音 enrollment 与 verification 必须发生在已验证会话内；任何语音失败都不能锁死 Passkey / SIWE 的可用路径。

  1. 服务器签发挑战

    Passkey ceremony 或 SIWE message 由服务器生成，保存 hash、过期时间与消费状态。

  2. 验证密码学凭证

    WebAuthn 校验 RP ID、origin、challenge、UP/UV；SIWE 严格校验 domain、URI、address、chain、nonce 与时间。

  3. 建立受控会话

    服务端签发 HttpOnly Cookie，登录后轮换 ID，支持退出、撤销、idle 与 absolute expiry。

  4. 获取单独同意

    显示用途、保存内容、保留期、撤回与替代路径；记录文本版本和地区。

  5. 短暂处理语音

    随机挑战先做质量、VAD、短语匹配，再做 speaker / PAD；原始音频只存在于隔离内存或 tmpfs。

  6. 输出分阶段评估

    VoiceAssessment 不使用含糊的单一 verified 状态；首版只观察或附加风控。

07 / Lifecycle

先定义删除，再定义采集

生物数据不可像密码一样更换。每一种数据都要在进入系统前明确保留期、撤销触发和备份边界。

建议默认值需要产品、隐私与法律共同确认。

08 / Threats

高风险路径必须有可测试控制

“监控”不是缓解措施。每个威胁都要落到实现、自动化测试、feature flag 或明确的产品边界。

发布前优先验证的威胁与控制。

09 / Timeline

六周依赖路径

每一周都有可观察的退出信号；未通过就不把风险转移到下一阶段。

  1. D1–D2Sprint 0 · 冻结与公开基线

    分支、CI、状态文案、契约示例、移动与 ARIA 修复、Playground / Docs 发布。退出：公网 200、CI 绿色、无认证误导。

  2. Week 1真实认证骨架

    契约、Auth API、PostgreSQL、Passkey、会话、安全头与限速。退出：预发真实 Passkey 会话可登录、刷新、退出和撤销。

  3. Week 2SIWE 与钱包绑定

    服务器 message / nonce、EOA/ERC-1271、重放/过期/拒签测试。退出：客户端伪造无效，同一签名重放必失败。

  4. Week 3语音垂直切片

    单独同意、服务器挑战、tmpfs、VAD、短语匹配、模型候选与加密模板。退出：错误短语失败，原始音频不进入持久化或日志。

  5. Week 4模型与隐私验证

    replay/TTS/噪声攻击集、EER/FMR/FNMR、切片评估、PIPIA/DPIA 与删除演练。退出：指标与已知失效环境可审计，否则 observe-only。

  6. Week 5安全与可靠性

    ASVS L2、扫描、故障注入、Runbook、备份恢复、密钥轮换和独立审查。退出：Critical / High 清零或有正式风险接受。

  7. Week 6邀请制发布

    20–50 名成年用户，10% canary 起步，语音默认 observe-only。退出：连续 7 天无 P0、无原始音频残留、SLO 达标。

11 / Gate

Go / No-Go 不是发布当天才看的清单

每项证据应在对应 Sprint 产生；发布负责人只汇总证据，不临时发明例外。

产品与真实性

  • 公网明确区分 Playground、Online Beta 与未来能力。
  • 生产路由无模拟音频、钱包、PIN 或假恢复。
  • 声音失败时仍可用强认证完成或恢复账户。

认证与会话

  • 每个 challenge 由服务器签发、过期且只能消费一次。
  • EOA / ERC-1271 正反测试齐全，SIWE 严格解析。
  • 会话只由服务器签发，可退出、撤销、超时和轮换。

声音与隐私

  • 错误短语失败；speaker 与 PAD 分数独立且可追溯。
  • 未过模型门时 VoiceAssessment 保持 observe-only。
  • 同意、撤回、删除、替代路径和影响评估已批准。
  • 异常路径也证明原始音频不进入持久化或备份。

工程与运维

  • CI、E2E、契约、迁移、安全扫描与预发冒烟绿色。
  • ASVS L2 有证据，Critical / High 为零或正式接受。
  • SLO、告警、Runbook、备份恢复、轮换与回滚演练通过。
  • feature flag 可关闭语音而不破坏强认证登录。
任何 P0 未通过，即判定 No-Go。声音、资产、隐私和会话安全不接受“先上线再补”的例外。
